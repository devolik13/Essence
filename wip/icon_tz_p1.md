# ТЗ для дизайна — Пакет 1: статусы, валюта, маркеры квестов

> Контекст: продолжение пакета аксессуаров и материалов (`accessories.svg`, `materials.svg`). Те же стилевые правила, тот же подход (один файл-спрайтшит на категорию, `<symbol>` внутри).

## Стилевая база (общая со всеми предыдущими паками)

- viewBox каждого `<symbol>`: **0 0 64 64** (мельче чем у оружия — иконки маленькие, нагружать деталями не нужно).
- Контур: `stroke="#1a1208"`, `stroke-width="1.4"`, `stroke-linejoin="round"`.
- Палитру градиентов и фильтров (`g_steel`, `g_brass`, `f_glow_red/blue/green` и т.д.) — **переиспользовать из `accessories.svg`**, можно скопировать `<defs>` оттуда.
- Стиль: плоский геральдический, с одним бликом и одной тенью, как в `armor.svg` / `weapons.svg`. Никакого реализма.
- Все 3 файла кладутся в `public/`: `public/statuses.svg`, `public/currency.svg`, `public/quests.svg`.

## Что нам потом сделать в коде (для справки)

После получения файлов я подключу:

```ts
// src/ui/weaponIcon.ts — добавить
export function spriteForStatus(id: string): SpriteRef | null {
  const map: Record<string, string> = {
    poison: 'st_poison', bleed: 'st_bleed', burn: 'st_burn', /* ... */
  };
  return map[id] ? { file: 'statuses.svg', id: map[id] } : null;
}

export function spriteForCurrency(tier: 'copper'|'silver'|'gold'): SpriteRef {
  return { file: 'currency.svg', id: `icon_coin_${tier}` };
}

export function spriteForQuestMarker(state: 'available'|'active'|'ready'|'main'): SpriteRef {
  return { file: 'quests.svg', id: `quest_${state}` };
}
```

И заменю текущие emoji-фолбэки в `UIScene.ts` (status icons), HUD-валюту, маркеры квестов над NPC.

## Что НЕ берём в этот пак

- Иконки достижений (~30 штук) — отдельный пак, после П1.
- Креатур-портреты для бестиария — большой арт, после.
- NPC-портреты для диалогов — после.

## Готовый формат отчёта дизайна

Три отдельных файла, каждый по схеме `accessories.svg`:
1. `<svg style="display:none">` обёртка с `<defs>` (можно скопировать из accessories.svg) и всеми `<symbol>` внутри.
2. Один файл = одна категория, не смешивать.
3. Класть в `public/`, я подключу.
